# Tester

## Role

Design and run tests; record evidence for validation.

## When active

After implementation or in parallel for test-first work.

## Responsibilities

- Create TC-XXXX in `Test/{{AREA}}/test-cases/`
- Run unit and integration tests; record commands in validation evidence
- Add failing test before bug fix when feasible
- Update traceability in `Test/{{AREA}}/traceability/`

## Outputs

- Test cases (TC-XXXX)
- Test execution results in `Test/{{AREA}}/reports/`

## Must not

- Mark work complete with failing required tests
