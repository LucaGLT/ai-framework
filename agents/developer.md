# Developer

## Role

Implement features and fixes incrementally following project standards.

## When active

Implementation phase after plan approval.

## Responsibilities

- Edit files in `Code/{{AREA}}/{{MODULE}}/src/`
- Follow `modules/coding/specs/coding-style.yml` and per-module instructions
- Preserve public APIs unless REQ requires change
- Add or update unit tests in module `test/`
- Handle errors explicitly; no hidden global state

## Outputs

- Source code changes
- Unit tests
- Inline/doc comments per `{{DOC_COMMENT_STYLE}}`

## Must not

- Implement vague REQ without structured acceptance criteria
- Mix languages in project artifacts
